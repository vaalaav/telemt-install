# telemt-install

Автоустановка **[telemt](https://github.com/telemt/telemt)** — Telegram MTProxy на Rust — на Ubuntu VPS, с интерактивным менеджером `mytelemtinfo`.

---

## Установка

Рекомендуемый способ (надёжно работает всегда):

```bash
curl -fsSL -o install.sh https://raw.githubusercontent.com/vaalaav/telemt-install/main/install.sh
chmod +x install.sh
sudo bash install.sh
```

Однострочный вариант (может не сработать с первого раза из-за CDN-кэша — тогда используйте способ выше):

```bash
sudo bash -c "curl -fsSL 'https://raw.githubusercontent.com/vaalaav/telemt-install/main/install.sh?v=$(date +%s)' | bash"
```

> Требуется root. Ubuntu 20.04 / 22.04 / 24.04, x86_64.

После установки управление — командой `sudo mytelemtinfo`.

---

## Сценарии установки

При запуске установщика первым шагом выбирается сценарий:

### 1. Стандартная установка
MTProxy на нескольких портах с маскировкой под чужие SNI (готовые пресеты или ваши собственные). Доступны все опции: фаервол, тюнинг сети, VLESS upstream, свой домен в ссылке клиента.

### 2. Свой сайт
Поднимает реальный сайт через nginx + Let's Encrypt на порту 8443, MTProxy остаётся на 443. В ссылке клиента — домен сайта (естественно выглядит для DPI). Сертификат настоящий, обновляется автоматически.

При выборе сценария **«Свой сайт»** запрашивается:
- Домен (с проверкой A-записи на сервер — обязательно)
- Email для Let's Encrypt
- URL шаблона сайта на GitHub (по умолчанию [`vaalaav/Market-Terminal-Template`](https://github.com/vaalaav/Market-Terminal-Template))

## Опции установки

В диалоге можно включить или пропустить любой компонент:

- **Инстансы telemt** — несколько пресетов или свой SNI и порт (только в сценарии «Стандартная»)
- **UFW + rate-limit** — фаервол с защитой от DPI-зондирования
- **TCP Keepalive** — фикс залипания iOS-клиентов после фона
- **BBR + fq qdisc** — congestion control для нестабильных сетей
- **nft SYN limiter** — per-client ограничение SYN на каждый порт
- **Тюнинг таймаутов** — для проблемных сетей (по умолчанию выключено)
- **`client_mss = "tspu"`** — обход ТСПУ (РФ-фильтрация), по умолчанию включён
- **Свой домен в ссылке** — вместо IP в `tg://proxy?server=` (только в сценарии «Стандартная»; в «Своём сайте» используется домен сайта)
- **VLESS Reality upstream** — туннелирует трафик telemt → Telegram DC через ваш VLESS-сервер. На выбор: одна `vless://` ссылка ИЛИ подписка 3x-ui (`https://...`) с балансировкой по нескольким узлам.

После финального подтверждения установка идёт автоматически. В конце — таблица статуса и готовые ссылки для клиентов.

## VLESS Reality upstream

Если Telegram DC блокируется на исходящем у вашего хостера, можно завернуть трафик через свой VLESS-сервер. Скрипт ставит `xray-core`, поднимает локальный SOCKS5 на `127.0.0.1:40000` и добавляет `[[upstreams]]` в конфиги telemt.

Поддерживаются два режима:

- **Одна `vless://` ссылка** — стандартный вариант для одного сервера
- **Подписка 3x-ui** (`https://server:port/path/sub-id`) — несколько узлов с балансировкой:
  - `leastPing` (по умолчанию) — автоматически выбирает быстрейший узел
  - `roundRobin` — по очереди
  - `random` — случайно
  - Опциональное **авто-обновление подписки** каждые 6 часов (systemd timer)

## Сайт-заглушка

Сценарий **«Свой сайт»** разворачивает на одном сервере и MTProxy, и реальный сайт:

```
ваш.домен:443   → telemt (MTProxy для клиентов Telegram)
ваш.домен:8443  → nginx (настоящий сайт с Let's Encrypt сертификатом)
ваш.домен:80    → nginx (только для ACME challenge)
```

В ссылке клиента: `tg://proxy?server=ваш.домен&port=443&secret=...`. Telegram-клиент идёт на 443 (там telemt). Браузер откроет `https://ваш.домен:8443/` и увидит ваш сайт с зелёным замком.

Шаблон сайта — любой GitHub-репозиторий со статикой. Меняется на лету через `mytelemtinfo` → 8.

## mytelemtinfo — управление после установки

```bash
sudo mytelemtinfo
```

8 разделов:

1. **Управление прокси** — старт/стоп/рестарт, добавление и удаление инстансов (до 10), ссылки клиентам, логи, **проверка работоспособности** (полный health check), полное удаление
2. **Сетевой тюнинг** — Keepalive + BBR
3. **nft SYN Limiter** — настройка параметров и счётчики дропов
4. **Настройки конфигов telemt** — таймауты `[timeouts]` + переключатель `client_mss = "tspu"` (вкл/выкл во всех конфигах)
5. **UFW / Rate-limit**
6. **Свой домен в ссылках** — с проверкой DNS (блокируется если установлен сайт-заглушка — домен берётся из сайта)
7. **VLESS Reality upstream** — изменить ссылку/подписку, обновить подписку сейчас, сменить стратегию балансировки, показать узлы, переключить авто-обновление, прицепить/отцепить от telemt, тест IP
8. **Сайт-заглушка** — установить, сменить шаблон, обновить сертификат, проверка работоспособности, логи nginx, удалить

### Автоматические проверки

После любого изменения через `mytelemtinfo` запускается **краткая проверка работоспособности** — статус всех инстансов, VLESS-туннеля, привязки upstream'ов, состояние сайта и сертификата. Полный health check доступен в подменю «Управление прокси» → пункт 10.

При обновлении бинарника telemt (пункт «Управление прокси» → «Обновить бинарник telemt») скрипт:
- Делает бэкап старого бинарника
- Сравнивает версии до/после
- Проверяет каждый конфиг через тестовый запуск (5 секунд с анализом journalctl на ошибки)
- При проблемах предлагает откатить к бэкапу

## Режимы запуска install.sh

При повторном запуске на уже установленной системе (или если найден telemt установленный другим способом) скрипт предлагает выбор:

- **Установка поверх** — оставляет существующие конфиги
- **Чистая установка** — полная очистка + новая установка
- **Только очистка** — удалить всё без новой установки
- **Только установить VLESS туннель** — поставить xray + VLESS upstream поверх существующего telemt (своего или чужого), автоматически прицепить ко всем найденным `/etc/telemt/telemt*.toml` или вывести инструкцию для ручной интеграции

Флаги командной строки:

```bash
sudo bash install.sh --update       # обновить только бинарник telemt
sudo bash install.sh --clean        # чистая установка без интерактивного выбора режима
sudo bash install.sh --purge        # только полная очистка
sudo bash install.sh --vless-only   # установить только VLESS туннель
```
